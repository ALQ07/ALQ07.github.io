## 资源加载与释放

在加载cocos中的动态资源时，我们一般会这样去使用：

```TypeScript
// 加载 SpriteFrame，image 是 ImageAsset，spriteFrame 是 image/spriteFrame，texture 是 image/texture
resources.load("test_assets/image/spriteFrame", SpriteFrame, (err, spriteFrame) => {
    this.node.getComponent(Sprite).spriteFrame = spriteFrame;
});
```

- resources 是内置的Bundle

由于resources.load是异步加载，一般我们还会使用promise进行一次封装：
```TypeScript
    load(path, callBack: (finished: number, total: number) => void): Promise<Asset> {
        return new Promise<Asset>((resolve, reject) => {
            GM.ResMgr.Res.load(path, (finished: number, total: number) => {
                callBack(finished, total);
            }, (error, asset) => {
                if (error) {
                    reject(error);
                } else {
                    resolve(asset);
                }
            });
        });
    }
```

但是，cocos在使用动态加载之后，会把加载的资源放到一个缓存中，不会被垃圾回收机制回收。如果我们不手动释放，那么缓存中的资源会一直存在，直到游戏结束，导致内存泄漏。
所以，我们在使用动态加载资源时，要注意手动释放资源，及时清理缓存。

### 释放资源

手动释放资源的方法有两种：

1. **引用计数**：

    当加载资源之后，使用 `addRef` 接口手动增加引用计数，增加引用计数后，可以保证该资源不会被提前错误释放。

   ```TypeScript
   resources.load('images/background/spriteFrame', SpriteFrame, function (err, spriteFrame) {
       self.getComponent(Sprite).spriteFrame = spriteFrame;
       spriteFrame.addRef();
   });
   ```

   而在不需要引用该资源以及相关组件，或者节点销毁时，请**务必记住**使用 `decRef` 移除引用计数，并将资源引用设为 `null`。

   ```TypeScript
   spriteFrame.decRef();
   spriteFrame = null;
   ```

2. **调用手动释放接口**：

   当我们加载的资源比较多，或者我们不确定什么时候才会不再需要使用该资源时，我们可以调用手动释放接口，手动释放资源。

   ```TypeScript
   assetManager.releaseAsset(texture);
   ```

2. Json格式压缩，就是去掉json文件中的空白、換行、无效字段。可以使用在线工具进行压缩，例如：[json压缩工具](https://www.bejson.com/compress/json/)，最后还能将json文件进行合并。

3. 二进制流加载，经过配置表优化、json压缩、json合并之后，再将json文件转换为二进制流加载。
